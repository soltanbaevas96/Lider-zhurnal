# Лидер+ · Система учёта уроков

Веб-система учёта работы преподавателей для учебного центра «Лидер+».
Учителя вносят проведённые уроки и прикрепляют планы, завуч видит часы по каждому
преподавателю, группе и ассистенту, выгружает отчёты в Excel.

**Стек:** React + Vite · Supabase (база, авторизация, хранилище файлов) · деплой на Vercel.

---

## Что уже умеет

- Вход по email/паролю с разделением ролей (преподаватель / завуч).
- Кабинет преподавателя: внесение урока (дата, время, группа, тема, ученики, статус,
  **ассистент**), прикрепление файла плана, **редактирование и удаление** своих уроков,
  свои часы за период.
- Кабинет завуча: сводка, список преподавателей, профиль каждого с разбивкой часов
  по группам, вкладка **Ассистенты** (занятия + часы + с какими преподавателями работал),
  редактирование/удаление любых уроков.
- **Фильтр по периоду**: месяц или произвольный диапазон дат. Экспорт в Excel
  (уроки + часы по учителям + лист ассистентов).
- **Ученики и посещаемость**: завуч заводит учеников и привязывает к группам (раздел
  «Управление» → «Ученики»). На проведённом уроке преподаватель отмечает посещаемость
  (по умолчанию все были, снимаешь отсутствующих). У завуча — вкладка «Посещаемость»
  с контролем по группам и по ученикам (процент, худшие сверху).
- **Управление справочниками из интерфейса** (завуч): добавление, редактирование и
  архивирование преподавателей, ассистентов, групп, предметов и учеников — без SQL.
- **Выдача доступа преподавателям из интерфейса**: завуч создаёт аккаунт входа и
  привязывает его к карточке преподавателя одной кнопкой (через защищённую Edge Function).
- Адаптивная вёрстка (ПК и телефон).

## Шаг 1. Создать проект Supabase

1. Зайдите на https://supabase.com → **New project**. Задайте имя, пароль БД, регион
   (ближе к Казахстану — например Frankfurt). Дождитесь готовности проекта.
2. Откройте **SQL Editor** → New query. Скопируйте и выполните целиком файл
   `supabase/01_schema.sql`. Это создаст таблицы, роли, политики доступа (RLS) и bucket
   для файлов планов.
3. Выполните `supabase/03_students_attendance.sql` — добавляет учеников, посещаемость,
   привязки групп/предметов к преподавателю и вход по логину. Безопасно на уже созданной базе.
4. (Необязательно, для демо-наполнения) выполните `supabase/02_seed.sql` — добавит
   предметы, группы, преподавателей, ассистентов и примеры уроков.

> Вход по логину: пользователи входят по логину (напр. `asaparova`), а не по email.
> Внутри Supabase Auth используется технический email `логин@lider.local` — его никто не
> видит. Первого администратора (завуча) создаём с логином, см. Шаг 2 ниже.

> ⚠️ Если у вас проект, созданный после мая 2026, может быть отключён автодоступ к новым
> таблицам через Data API. Если запросы возвращают пустоту/ошибку доступа — зайдите в
> **Project Settings → Data API** и включите доступ к таблицам схемы `public`, либо в SQL
> выполните `grant select, insert, update, delete on all tables in schema public to anon, authenticated;`

## Шаг 2. Создать первого пользователя (завуча)

Первого администратора создаём вручную. Вход в систему — по логину, поэтому email
задаём в техническом формате `логин@lider.local`.

1. **Authentication → Users → Add user → Create new user.**
   - **Email**: `zavuch@lider.local` (это технический email, входить будете по логину `zavuch`)
   - **Password**: придумайте, запишите
2. Сделать его админом и задать логин — **SQL Editor → New query → Run**:
```sql
update profiles set role = 'admin', username = 'zavuch' where id =
  (select id from auth.users where email = 'zavuch@lider.local');
```
3. На сайте входите: логин `zavuch` + ваш пароль.

**Преподавателей** дальше заводит завуч через интерфейс: раздел «Управление» →
вкладка «Преподаватели» → «Выдать доступ». Логин предлагается автоматически из ФИО
(первая буква имени + фамилия транслитом), пароль генерируется. Для этого нужно один
раз развернуть Edge Function (Шаг 2б).

> Только первого админа создаём вручную — назначить его больше некому. Всё остальное
> (преподаватели, их логины, группы, предметы, привязки) делается в интерфейсе.

## Шаг 2б. Развернуть Edge Function для выдачи доступа

Создание аккаунтов требует сервисного ключа, который нельзя держать в браузере.
Поэтому это делает серверная функция `invite-teacher`.

Установите Supabase CLI (https://supabase.com/docs/guides/cli), затем:

```bash
supabase login
supabase link --project-ref ВАШ_PROJECT_REF
supabase functions deploy invite-teacher
```

`SUPABASE_URL`, `SUPABASE_ANON_KEY` и `SUPABASE_SERVICE_ROLE_KEY` доступны функции
автоматически как встроенные секреты проекта — дополнительно ничего задавать не нужно.

Если CLI не используете — можно создать функцию через дашборд: **Edge Functions → Deploy
a new function**, имя `invite-teacher`, вставить код из
`supabase/functions/invite-teacher/index.ts`.

## Шаг 3. Настроить переменные окружения

1. Скопируйте `.env.example` в `.env.local`.
2. В Supabase: **Project Settings → API** скопируйте `Project URL` и `anon public` ключ.
3. Впишите их:
```
VITE_SUPABASE_URL=https://ваш-проект.supabase.co
VITE_SUPABASE_ANON_KEY=ваш-anon-ключ
```

```bash
npm install
npm run dev
```
Откройте http://localhost:5173

## Шаг 5. Деплой на Vercel

1. Залейте проект в репозиторий GitHub.
2. На https://vercel.com → **New Project** → импортируйте репозиторий.
3. Framework: **Vite** (определится автоматически).
4. В **Environment Variables** добавьте `VITE_SUPABASE_URL` и `VITE_SUPABASE_ANON_KEY`.
5. **Deploy**.

## Модель данных (кратко)

- **profiles** — пользователи с ролью (`teacher` / `admin`), связаны с `auth.users`.
- **teachers** — преподаватели; `profile_id` связывает с аккаунтом (может быть пустым).
- **assistants** — общий список ассистентов центра.
- **groups**, **subjects** — справочники.
- **lessons** — уроки: преподаватель, группа, **ассистент (опц.)**, дата, время, тема,
  ученики, статус, путь к файлу плана.

Доступ разграничен на уровне базы (RLS): преподаватель видит и меняет только свои уроки,
завуч — всё. Справочниками управляет только завуч.
